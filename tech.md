# Тестовое задание: Интернет-магазин

## Описание проекта

Небольшая торговая компания хочет запустить онлайн-магазин. Вам необходимо разработать full-stack веб-приложение на версии не ранее чем PHP 8.3 + Laravel 11 (backend) и Vue.js 3 (frontend), которое позволит покупателям заказывать товары, а администраторам — управлять каталогом и заказами.

## Основной функционал

### Для покупателей:
- Регистрация и авторизация (email + пароль)
- Просмотр каталога товаров с фильтрацией (минимум по категории) и поиском
- Просмотр детальной информации о товаре
- Добавление товаров в корзину и оформление заказа
- Просмотр списка своих заказов

### Для администраторов:
- Отдельная панель администратора (роль admin в БД)
- Управление товарами (создание, редактирование, удаление)
- Просмотр списка заказов с возможностью изменения статуса

## Обязательные технические требования

1. Backend: от PHP 8.3, Laravel 11
2. Frontend: Vue.js 3 (Composition API обязательно)
3. Архитектура: SPA с REST API или использование Inertia.js (на ваш выбор)
4. База данных: PostgreSQL или MySQL
5. Очереди: Использование Laravel Jobs/Queues хотя бы для одной задачи (например, отправка email при создании заказа - можно использовать log driver)
6. Backend архитектура: Выделение бизнес-логики из контроллеров (Services или Actions)
7. API: REST API для основных операций
8. Валидация: Form Requests на backend
9. Миграции и Seeders: Миграции для всех таблиц, seeders с тестовыми данными (минимум 10-15 товаров, 1 admin, 1-2 обычных пользователя)

## Обязательные возможности Vue.js части

- Реактивный интерфейс корзины
- Минимум 5-7 переиспользуемых компонентов
- Vue Router для навигации
- Обработка loading states при запросах к API
- Базовая обработка ошибок

## Дополнительные требования (опционально, но будет плюсом)

Реализуйте 1-2 пункта на выбор, чтобы продемонстрировать свои сильные стороны:

- Использование Pinia для управления состоянием
- Кеширование списка товаров в Redis
- Events & Listeners для отправки уведомлений
- Rate Limiting для API
- Загрузка и хранение изображений товаров
- TypeScript для Vue.js части
- Docker Compose для запуска проекта

## Минимальная структура БД

Спроектируйте самостоятельно, но должно быть минимум:
- Пользователи (с ролями)
- Товары (с категориями)
- Заказы (со связью с пользователями)
- Позиции заказов (связь заказ-товар с количеством)

## Что мы будем оценивать

### Backend:
- Чистоту архитектуры и разделение ответственности
- Эффективность работы с БД (избегание N+1)
- Правильность использования возможностей Laravel

### Frontend:
- Структуру компонентов и их переиспользуемость
- Работу с Composition API
- UX (loading states, обратная связь)

### Общее:
- Читаемость и качество кода
- Документацию (README с инструкциями)
- Git commits (осмысленные сообщения)

- Не требуется идеальный дизайн — достаточно простой, функциональный интерфейс
- Не нужны все дополнительные фичи — выберите 1-2, где чувствуете себя сильнее
